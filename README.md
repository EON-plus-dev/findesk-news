# FinDesk

Публічний бізнесовий опис поточного функціоналу FinDesk.

Цей репозиторій не призначений для коду, технічної архітектури, внутрішніх інструкцій, доступів, токенів, паролів, ключів, логів або клієнтських даних. Тут має бути тільки корисна для бізнесу інформація: що вже є у продукті, для кого це працює, які сценарії покриває, які інтеграції підтримуються і де є межі поточного функціоналу.

Стан опису: 19 червня 2026 року.

## 1. Що таке FinDesk

FinDesk - це робоча платформа для українських ФОП, бухгалтерів, бухгалтерських аутсорсерів, компаній та власників бізнесу. Продукт об'єднує кабінети бізнесу, книгу доходів, платежі, податки, звітність, документи, контрагентів, працівників, електронний документообіг, листи ДПС, перевірки, нагадування, інтеграції та AI-помічника.

Головна бізнесова ціль: не вести облік, документи, податки і контроль ризиків у різних таблицях, кабінетах і чатах, а мати один робочий простір, де видно стан бізнесу та наступні дії.

## 2. Для кого продукт

### ФОП

FinDesk покриває базові щоденні задачі ФОП:

- ведення книги доходів;
- контроль ліміту доходу;
- підготовку декларацій і звітів;
- податковий календар і нагадування;
- податкові платежі;
- документи, рахунки, акти, договори;
- листи та повідомлення від ДПС;
- моніторинг перевірок;
- роботу з банківськими та платіжними джерелами;
- роботу з бухгалтером або консультантом у спільному кабінеті.

### Бухгалтер і бухгалтерський аутсорсер

FinDesk допомагає вести багато клієнтів одночасно:

- бачити список кабінетів і статус кожного;
- швидко знаходити кабінети з проблемами, дедлайнами або перевірками;
- контролювати, де не підтягнулись інтеграції;
- формувати звіти та перевіряти дані перед поданням;
- працювати з документами клієнтів;
- отримувати сповіщення по податкових подіях, листах ДПС і перевірках;
- працювати з групою ФОП як з портфелем.

### ТОВ і бізнес-команди

Для компаній FinDesk покриває:

- фінансові операції;
- продажі та рахунки;
- складський облік у бізнесовому інтерфейсі;
- HR / Payroll;
- податкову та управлінську звітність;
- електронний документообіг;
- командні ролі;
- контрагентів, реквізити, документи і платежі.

### Фізичні особи

Для фізичних осіб у продукті передбачені сценарії:

- декларація про майновий стан і доходи;
- декларація для податкової знижки;
- облік майна: нерухомість, земля, транспорт;
- пов'язані платежі;
- AI-пояснення нарахувань.

| ТОВ | Фінанси, HR / Payroll, продажі, склад, звіти, платежі, перевірки, документи, команда, ролі, КЕП | Директор, головний бухгалтер, бухгалтер, HR, менеджер продажів, юрист, комірник, логіст, аудитор |
| Група ФОП | Зведений реєстр документів по ФОП, список ФОП у групі, масові дії, зведені звіти, перевірки по групі | Аутсорсер, адміністратор групи, бухгалтер групи |

## 4. Кабінети бізнесу

Кабінет - це центр роботи з конкретним ФОП, компанією, фізичною особою або групою ФОП.

У кабінеті ведеться:

- назва, тип і роль користувача;
- податковий профіль;
- КВЕДи та ліцензійні параметри;
- реквізити для документів;
- банківські дані;
- КЕП і статуси підписів;
- джерела даних та підключення;
- команда і ролі;
- сповіщення;
- довідники: контрагенти, категорії, номенклатура, основні засоби.

Для бізнесу це означає, що кожен клієнт або кожна організація має власний контекст: свої податки, документи, інтеграції, строки, команду та ризики.

## 5. Мультикабінет і портфель клієнтів

Мультикабінет потрібен бухгалтеру або власнику групи бізнесів.

Поточне покриття:

- список кабінетів;
- фільтри за типом, роллю і статусом;
- підсумок по кількості кабінетів;
- загальний дохід;
- кабінети, які потребують уваги;
- найближчі дедлайни;
- швидкий перехід у конкретний кабінет;
- окремі сценарії для групи ФОП: документи групи, ФОП у групі, масові дії, зведені звіти, перевірки.

Для аутсорсера це не просто список клієнтів, а робоча черга: де є дедлайн, де не підписано документи, де помилка інтеграції, де ризик перевірки.

## 6. Команда, ролі та доступи

FinDesk підтримує командну роботу в межах кабінету.

Поточні ролі за типами кабінетів:

| Тип кабінету | Ролі |
|---|---|
| ТОВ | Директор, головний бухгалтер, бухгалтер, HR/Payroll, менеджер з продажу, юрист, комірник, логіст, аудитор |
| ФОП | Власник, бухгалтер, аудитор |
| Група ФОП | Адміністратор групи, бухгалтер групи, перегляд |
| Фізична особа | Власник, податковий консультант |

Учасники можуть мати статус активного користувача, запрошеного користувача або неактивного користувача. Це дає змогу працювати над одним кабінетом кільком людям, не змішуючи відповідальність власника, бухгалтера, аудитора чи менеджера.

## 7. Налаштування кабінету

У налаштуваннях кабінету доступні такі блоки:

- профіль та реквізити;
- податковий профіль;
- КВЕДи та ліцензування;
- КЕП / підпис;
- підключення та синхронізація;
- еквайринг;
- сповіщення;
- команда;
- довідники.

Для пасивних кабінетів доступ обмежений: реквізити, КЕП для підписання, базові підключення, сповіщення і довідник контрагентів.

## 8. КЕП і підписання

FinDesk веде інформацію про електронні підписи у кабінеті.

Поточне покриття:

- власник підпису;
- посада або роль;
- центр видачі;
- період дії;
- статус: дійсний, скоро завершується, прострочений;
- сценарії підписання документів;
- масовий підпис документів у групі ФОП;
- використання підпису для звітності та документообігу.

Приклади центрів і каналів, які відображаються у продукті: АЦСК ІДД ДПС, ПриватБанк, Дія.Підпис.

У цьому публічному описі навмисно не публікуються жодні деталі зберігання, передачі або обробки ключів.

- Нова Пошта;
- Checkbox;
- Cashalot;
- Poster POS;
- OData 1С;
- Дія.Бізнес як окремий marketplace-сценарій;
- ПУМБ Бізнес як доступне користувачу банківське підключення.

Вони можуть бути в дослідженні, частково підготовлені або зарезервовані для майбутніх сценаріїв, але в цьому README не заявляються як повноцінний поточний функціонал.

## 10. Книга доходів

Книга доходів - один із ключових модулів для ФОП.

### Що бачить користувач

- записи за датами;
- опис операції;
- контрагента;
- ЄДРПОУ або ІПН контрагента;
- IBAN контрагента;
- суму операції;
- суму, яка потрапляє до книги доходів;
- тип оплати;
- джерело операції;
- статус операції;
- пов'язаний документ;
- комісію платіжної системи;
- ручні та автоматичні категорії;
- AI-пояснення або уточнення.

### Типи оплат

- готівка;
- банк;
- картка;
- ПРРО/РРО як тип операції у моделі обліку.

### Джерела операцій

- Monobank;
- ПриватБанк / Приват24;
- Way4Pay;
- LiqPay;
- Vchasno;
- ручне внесення;
- ПРРО як модельний тип джерела, без заяви про повну готовність Checkbox/Cashalot у публічному описі.

### Статуси записів

- дохід;
- не дохід;
- витрата;
- внутрішній переказ;
- повернення;
- потребує уточнення.

### Які проблеми система підсвічує

- відсутнє призначення платежу;
- невідомий контрагент;
- невизначена класифікація;
- відсутній документ;
- можливий дублікат;
- підозріла сума;
- розбіжність із джерелом.

### Аналітика книги

Книга агрегує дані по місяцях і показує:

- готівкові надходження та повернення;
- банківські та карткові надходження;
- ПРРО/РРО-надходження у моделі книги;
- загальний дохід;
- загальні повернення;
- суму, що входить до книги;
- кількість записів;
- наявність проблемних записів.

## 11. Контроль ліміту ФОП

FinDesk відстежує використання ліміту доходу ФОП.

Поточне покриття:

- сума доходу з книги;
- відсоток використання ліміту;
- попередження при наближенні до порогів;
- сценарний аналіз у аналітиці;
- зв'язок із групою ФОП і портфельним переглядом;
- використання даних книги доходів для податкових рішень.

Бізнесова цінність: бухгалтер або власник бачить ризик перевищення ліміту до того, як це стане податковою проблемою.

FinDesk має уніфікований простір платежів.

### Типи платежів

- надходження;
- податок;
- податок ФОП;
- податок із зарплати;
